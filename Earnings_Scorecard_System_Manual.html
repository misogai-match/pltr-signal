02

「PLTRシグナル」コマンドの裏側

ユーザーが「PLTRシグナル」と入力すると、Claudeはバックグラウンドで以下を実行します。

1
Web検索 — リアルタイムデータ取得 PLTRの現在株価、前日比、時価総額を取得。同時にCNN Fear & Greed Indexの最新値と、S&P 500やVIXなどマーケット環境も確認。
2
Yahoo Finance Analysis — コンセンサス最新値取得 Yahoo Finance の Analysis ページから最新の EPS Estimate / Revenue Estimate を取得。T-90d 初期値（EPS $0.28 / Rev $1.54B）との差分を Consensus Drift として自動計算。株価急騰・急落時にはアナリストの予想修正が入ることが多く、毎回取得することでドリフトの動きをリアルタイムに捕捉する。
3
QoQ株価変動率の計算 現在株価 ÷ 前回決算日終値 - 1 = QoQ変動率。この値がバリュエーション補正（Tier 0）の入力になる。例: $129 ÷ $147.76 = -12.7%
4
スコア計算 Tier 2（トレンド）+ Tier 3（品質）は前回決算で確定済みの値を使用。補正①（バリュエーション）と補正②（マクロF&G）を最新データで再計算。Consensus Driftも最新値で更新。Tier 1のEPS/Rev Beat%のみ決算発表まで「待ち」。
5
バリュエーション判定 現在株価とBase理論株価（$147）の乖離率から、「割安〜大幅割高」の5段階ゾーンを判定。
6
ダッシュボード生成（3形式同時出力） JSX（Reactコンポーネント）、HTML（自己完結型）、PNG（Playwrightスクリーンショット）を同時生成。データは全てハードコードされ、外部API不要で表示可能。
7
バージョニング＆出力 パッチバージョン+1、Changelog記録。ファイル名: PLTR_LiveSignal_vX.X.X_YYYYMMDD。変更内容サマリーを添えてユーザーに納品。
ユーザーの作業
HTMLをダウンロード → GitHubにドラッグ&ドロップ → 自動公開。所要時間: 約2分。

04

スコアリングロジック（3-Tier + 補正）

全体構造

Total Score = Tier1 (40pt) + Tier2 (35pt) + Tier3 (25pt) + 補正① (±15pt) + 補正② (±5pt) 範囲: 理論上 -20 〜 120 / 実用上 0 〜 100

Tier 1: サプライズ指標 40pt

決算発表時に初めて判明するデータ。Pre-Earningsシグナルでは「?」表示。

指標配点ロジック
EPS Beat/Miss % 15pt Actual EPS ÷ Consensus EPS - 1
+10%以上 → 15pt、Miss → 0pt、段階配点
Revenue Beat/Miss % 15pt Actual Rev ÷ Consensus Rev - 1
+3%以上 → 15pt、Beat幅に応じた段階配点
Consensus Drift 10pt シグナル生成時にYahoo Finance Analysisから最新Estimateを自動取得し、T-90d初期値との乖離率を算出。上方修正 → 加点、下方修正 → 減点

Tier 2: トレンド加速 35pt

前四半期からの「加速度」を測定。絶対値ではなく変化の方向が重要。

指標配点ロジック
NDR QoQ変化 10pt Net Dollar Retention の前期比変化
上昇 → 加点、低下 → 減点
Revenue 加速度 10pt Revenue YoY Growth の QoQ差分
加速 → 加点、減速 → 減点
Margin QoQ改善 10pt Adj. Operating Margin の前期比改善幅
改善 → 加点、悪化 → 減点
US Commercial YoY 5pt US Commercial Revenue の YoY成長率
高成長 → 加点

Tier 3: 構造品質 25pt

ビジネスの「質」を評価。一時的な好決算と構造的な強さを区別する。

指標配点ロジック
SBC % Revenue 8pt SBC ÷ Revenue。低いほど株主フレンドリー
15%以下 → 8pt、30%超 → 0pt
FCF Margin 8pt Adj. FCF ÷ Revenue。キャッシュ創出力
30%超 → 8pt、マイナス → 0pt
Rule of 40 9pt Revenue YoY% + FCF Margin%
80超 → 9pt、40未満 → 0pt

補正①: バリュエーション過熱度 ±15pt

決算前のQoQ株価変動率で過熱/売られすぎを補正。

QoQ = 現在株価 ÷ 前回決算日終値 - 1 +40%超 → -5pt（過熱警戒） +20〜40% → -3pt -20〜+20% → 0pt（中立） -40〜-20% → +3pt -40%以下 → +5pt（売られすぎ）
なぜ必要か？
Q1'25の教訓：全KPI好調で60pt級でも、QoQ+40%超の過熱状態では-9%下落。バリュエーション補正なしでは過熱シグナルを捉えられない。

補正②: マクロF&G ±5pt

CNN Fear & Greed Index（T-7d値）で市場センチメントを反映。

F&G ≤ 24（Extreme Fear）→ +5pt F&G 25-44（Fear） → +2pt F&G 45-55（Neutral） → 0pt F&G 56-74（Greed） → -2pt F&G ≥ 75（Extreme Greed）→ -5pt

シグナル判定

スコアシグナル意味
55+強気 BULLISH高確率で好決算 → 積極的にポジション構築
40-54やや強気 LEAN BULL好材料が優勢 → 決算内容を見て追加判断
25-39中立 NEUTRAL判断材料不足 → 静観
<25やや弱気 LEAN BEAR悪材料が優勢 → ポジション縮小を検討
05

バリュエーションモデル

2つのアプローチを組み合わせて多面的に評価します。

① Reverse DCF — 市場が織り込む成長率の逆算

問い: 現在のEVを正当化するには、何年間何%の成長が必要か？
【前提】 WACC（割引率）: 10% Terminal Growth Rate: 3% Terminal EV/Revenue Multiple: 12x Projection Period: 7年 【逆算】 ターミナル年必要Revenue = EV × (1+WACC)^7 ÷ Terminal Multiple ★織込みCAGR = (ターミナルRev ÷ 現在Rev)^(1/7) - 1 【判定】 織込みCAGR > ガイダンス → 市場が過剰な成長を期待 → 割高圧力 織込みCAGR < ガイダンス → 成長を十分評価していない → 割安余地

② FCF Fair Value — 3シナリオ理論株価

FCFの成長フェーズを3段階に分けて将来キャッシュフローを投影。

BearBaseBull
Phase 1 (3年)30%40%50%
Phase 2 (2年)18%25%35%
Phase 3 (1年)12%18%25%
Terminal Multiple22x28x35x
★ 理論株価$85$147$263
Fair Value = (Σ FCF_t / (1+WACC)^t + Terminal Value) / 発行株式数 + Cash per Share

追加指標

指標算出方法意図
Forward P/E 株価 ÷ FY27E Adj EPS
FY27E = FY26 Adj OI × (1+Base Phase1成長率) ÷ 発行株式数		 Trailing P/Eだけでは割高に見えすぎる。FY27E（finviz慣行）で圧縮ストーリーを可視化
PEG Ratio Forward P/E ÷ FY25→FY27E EPS CAGR% 成長率対比の割安度。ただし高成長期はCAGRが大きくPEGが人工的に低くなるため要注意
FCF Yield FY26E FCF ÷ 時価総額 キャッシュフローベースの利回り

バリュエーション・ゾーン判定

毎回更新されるもの vs 四半期でのみ更新されるもの

理論株価（Bear $85 / Base $147 / Bull $263）は四半期決算ごとに更新されます。DCFモデルの入力値（ガイダンスFCF、成長率前提、Cash、発行株式数）が決算発表で変わるまでは固定です。

一方、「PLTRシグナル」を実行するたびに毎回更新されるのは以下です:
毎回更新（リアルタイム）四半期決算で更新
現在株価 → QoQ変動率
F&G Index → マクロ補正
コンセンサス（EPS/Rev Est.）→ Drift
乖離率（株価 vs Base）→ ゾーン判定 		FY Guided FCF（DCF入力）
Phase成長率前提（必要時に見直し）
Terminal Multiple（同上）
Cash / 発行株式数（BS更新）
→ 理論株価が再計算される
つまり、シグナルを途中で見た場合は「固定のBase $147に対して、今の株価がどこにいるか」が動く形です。決算間でBase前提を見直したい場合は、Claudeに依頼すればモデルを再計算できます。

> +20%割高赤
> -10%適正灰

バックテストと誤判定分析

88%
方向一致率（14/16四半期）
16Q
バックテスト期間（Q1'22〜Q4'25）

Y値の定義

Y = 翌営業日始値 ÷ 決算日終値 - 1

決算発表後の市場初動反応を捉える。+なら好反応、-なら悪反応。

判定基準

記号条件
◎ 的中シグナル方向とY値の方向が一致（強気→+、弱気→-）
○ 概ね的中方向は一致するが小幅（Y値の絶対値が3%未満）
× 誤判定シグナルと逆方向の結果

誤判定の構造分析

Q4'22: やや弱気 → +14.4%（×）
GAAP黒字化転換という「レジーム変化」が発生。後ろ向きKPIでは構造変化を捉えられない。
→ 教訓: 定性判断（GAAP転換等）の併用が必要
Q1'25: 強気 → -8.7%（×）
全KPI好調（Tier1-3で60pt級）だが、FYガイダンスが市場期待を下回った。
→ 教訓: ガイダンス変数の追加が必要（P1タスク）

他銘柄への横展開ガイド

そのまま使える要素

要素汎用性備考
Tier 1: サプライズ高EPS/Rev Beat%とConsensus Driftはどの銘柄でも適用可
Tier 3: 構造品質高SBC%, FCF Margin, Rule of 40 は SaaS全般で有効
補正①: バリュエーション高QoQ株価変動率はユニバーサル
補正②: マクロF&G高市場全体のセンチメント。銘柄非依存
Reverse DCF高前提パラメータを変えるだけ
FCF Fair Value高Phase成長率とTerminal Multipleを銘柄ごとに設定
ダッシュボード高データ差し替えだけで別銘柄に転用可

銘柄ごとにカスタマイズが必要な要素

要素理由対応方針
Tier 2: トレンド加速 NDRやUS Commercialは PLTR固有KPI 銘柄のキードライバーに置換。例: AWS → リージョン数、NVDA → データセンター売上
Forward P/E 算出基準 Adj OI vs GAAP NI の選択は企業のプロファイルによる 黒字企業はGAAP EPS、高成長は Adj ベース
スコア閾値 ボラティリティの大小で反応幅が異なる バックテストN≥8で閾値調整

横展開の手順

1
対象銘柄のKPI特定 IR資料を読み、株価を動かす3-5個のキードライバーを特定。Tier 2の4指標をこれに置換。
2
過去8Q以上のデータ収集 Yahoo Finance（コンセンサス）+ IR資料（KPI）+ 10-Q（財務）から時系列データを構築。
3
バックテスト実施 Tier配点を仮設定 → 8Q以上でバックテスト → 方向一致率75%以上を目指す → 閾値調整。
4
バリュエーションモデル構築 Excelテンプレートをコピー → 株価・財務データ・成長率前提を差し替え → DCF自動再計算。
5
ダッシュボード生成 Claudeに「[ティッカー]シグナル」→ 株価・F&G・Yahoo Financeコンセンサスを自動取得し、データ差し替え版を生成 → GitHub Pagesで公開。
推定所要時間（2銘柄目以降）
KPI特定: 1時間 → データ収集: 2-3時間 → バックテスト+調整: Claude と 1セッション → ダッシュボード: 自動生成
初回（PLTR）で仕組みを構築済みなので、2銘柄目からは大幅に速くなります。
